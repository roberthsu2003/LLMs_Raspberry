# 目錄操作練習 – 學生實作手冊

歡迎來到目錄操作實作練習！請在您的 Raspberry Pi（或任何 Linux 終端）上逐一完成以下任務。每完成一個任務，請在終端機執行相關指令確認結果，並將螢幕輸出保存作為作業回報的依據。

---

## 📋 練習前準備

在開始之前，請先確認您的環境：
```sh
# 檢查目前位置
pwd

# 檢視環境資訊
whoami
uname -a
```

## 1️⃣ 基礎定位練習 – 找到你的位置

### 任務目標
熟悉如何確認目前位置並了解目錄結構。

### 操作步驟
1. 打開終端機，使用 `pwd` 確認您目前所在的目錄
2. 輸入 `ls -a`，列出當前目錄下所有檔案（包括隱藏檔）
3. 複製當前路徑（例如 `/home/pi`），貼到筆記本中保存

### 預期結果
- 清楚知道自己的起點位置
- 了解當前目錄中有哪些檔案和子目錄

### 自我檢查
- [ ] 我能說出目前所在目錄的完整路徑
- [ ] 我能識別隱藏檔（以 . 開頭的檔案）
- [ ] 我了解家目錄的位置

---

## 2️⃣ 目錄建立與導航 – 建立你的練習空間

### 任務目標
學會建立多層目錄結構並熟練地在目錄間移動。

### 操作步驟
1. 使用 `mkdir -p practice/step1/step2` 一次建立三層目錄
2. 用 `cd practice/step1/step2` 進入最底層目錄
3. 再次執行 `pwd`，確認您已在 `step2` 目錄
4. 使用 `cd ..` 回到 `step1`，檢查 `pwd` 是否正確
5. 用 `cd ../../` 回到 `practice` 目錄
6. 最後回到家目錄：`cd ~`

### 預期結果
- 成功建立三層目錄結構
- 能夠自如地在不同層級目錄間移動
- 理解相對路徑和絕對路徑的差異

### 自我檢查
- [ ] 我能使用 `mkdir -p` 建立多層目錄
- [ ] 我了解 `cd ..` 和 `cd ../../` 的差異
- [ ] 我能快速回到家目錄

## 3️⃣ 檔案篩選與識別 – 找出特定檔案

### 任務目標
學會使用萬用字元篩選特定類型的檔案。

### 操作步驟
1. 在 `practice` 目錄下，建立以下檔案和目錄：
   ```sh
   touch fileA.txt fileB.log fileC.md
   mkdir dirA dirB
   ```
2. 用 `ls *.txt` 只顯示 `.txt` 檔案
3. 用 `ls -d */` 列出所有子目錄（不顯示子目錄內的內容）
4. 用 `ls -l` 確認檔案與目錄的權限與擁有者

### 預期結果
- 能夠精準篩選出特定類型的檔案
- 理解萬用字元 `*` 的作用
- 能夠區分檔案和目錄的顯示方式

### 自我檢查
- [ ] 我能使用 `*.txt` 找出所有文字檔
- [ ] 我了解 `ls -d */` 的作用
- [ ] 我能從 `ls -l` 輸出中讀取檔案資訊

## 4️⃣ 目錄清理 – 安全刪除練習

### 任務目標
學會安全地刪除目錄和清理工作空間。

### 操作步驟
1. 使用 `rmdir dirA` 刪除剛才建立的空目錄 `dirA`
2. 嘗試直接 `rmdir practice`（應該會失敗，因為 `practice` 內仍有檔案）
3. 觀察錯誤訊息，理解為什麼會失敗
4. 用 `rm fileA.txt fileB.log fileC.md` 刪除所有檔案
5. 用 `rmdir dirB` 刪除另一個空目錄
6. 再次嘗試 `rmdir practice`，確認目錄已成功刪除

### 預期結果
- 理解 `rmdir` 只能刪除空目錄的限制
- 學會先清理檔案再刪除目錄的步驟
- 能夠安全地清理工作空間

### 自我檢查
- [ ] 我理解為什麼第一次 `rmdir practice` 會失敗
- [ ] 我知道如何正確地刪除含有檔案的目錄
- [ ] 我能安全地清理練習用的目錄

## 5️⃣ 進階練習：使用 tree 指令

### 任務目標
學會使用 tree 指令檢視目錄結構。

### 操作步驟
1. 如果系統尚未安裝 `tree`，先執行：
   ```sh
   sudo apt update && sudo apt install -y tree
   ```
2. 在任意目錄（例如 `$HOME`）執行 `tree -L 2`，觀察兩層深度的樹狀結構
3. 嘗試 `tree -d -L 1` 只列出目錄名稱，不顯示檔案
4. 在 `practice` 目錄下執行 `tree`，觀察完整的目錄結構

### 預期結果
- 能夠以樹狀圖方式檢視目錄結構
- 了解 `tree` 指令的各種參數用法
- 能夠快速了解複雜的目錄層級

### 自我檢查
- [ ] 我能使用 `tree` 指令顯示目錄結構
- [ ] 我了解 `-L` 和 `-d` 參數的作用
- [ ] 我能解讀 `tree` 輸出的目錄層級

## 6️⃣ 綜合挑戰 – 建立專案目錄結構

### 任務目標
綜合運用所學技能，建立一個完整的專案目錄結構。

### 操作步驟
1. 建立一個名為 `my_project` 的目錄
2. 在 `my_project` 中建立以下子目錄：
   ```sh
   mkdir -p my_project/{src,docs,tests,assets/images,assets/css,assets/js}
   ```
3. 在 `src` 中建立 `main.py`，寫入以下內容：
   ```python
   #!/usr/bin/env python3
   print("Hello, Linux!")
   ```
4. 用 `chmod +x src/main.py` 讓它變成可執行檔
5. 使用 `./src/main.py` 執行，確認螢幕顯示 `Hello, Linux!`
6. 用 `tree -L 3 my_project` 檢查整個專案結構

### 預期結果
- 成功建立複雜的多層目錄結構
- 建立可執行的 Python 腳本
- 能夠使用樹狀圖檢視完整的專案結構

### 自我檢查
- [ ] 我能建立複雜的目錄結構
- [ ] 我了解如何建立可執行檔
- [ ] 我能驗證專案結構的正確性
