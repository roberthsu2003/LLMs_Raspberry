## 語法解剖：拆解一個 Linux 指令

在終端機中，一個完整的指令通常由三個部分組成：**命令 (Command)**、**選項 (Options)** 和 **參數 (Arguments)**。

```sh
# 指令結構示意圖
┌──────────┐   ┌───────────────┐   ┌────────────────┐
│   命令   │   │     選項      │   │      參數      │
└──────────┘   └───────────────┘   └────────────────┘
      ls              -la             /home/user
```

#### 1. 命令 (Command)
- **作用**：您希望電腦執行的動作，是指令的核心，不可或缺。
- **範例**：`ls` (列出目錄內容), `cd` (切換目錄), `rm` (刪除檔案)。

#### 2. 選項 (Options / Flags)
- **作用**：用來微調命令的行為，如同功能的「開關」。
- **格式**：
    - **短格式**：一個減號 (`-`) 接單個字母，如 `-l`, `-a` (可合併為 `-la`)。
    - **長格式**：兩個減號 (`--`) 接完整單字，如 `--all`, `--human-readable`。
- **範例**：`ls -l` 中的 `-l` 會讓列表以詳細格式顯示。

#### 3. 參數 (Arguments)
- **作用**：命令要操作的「對象」，例如檔案、目錄路徑或文字。
- **範例**：`cd /home/user` 中的 `/home/user` 就是 `cd` 命令要切換到的目標目錄。

#### 綜合範例
讓我們再次解析這個指令：`ls -la /home/user`
- **命令**: `ls` (我想「列出」)
- **選項**: `-la` (我想「詳細地」並且「列出全部」)
- **參數**: `/home/user` (我想列出的「對象」是這個路徑)

理解這個結構，您就能更快地學會看懂 `man` 手冊並掌握新指令！
