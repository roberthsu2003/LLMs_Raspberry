# 函式和工具-function calling
open-webui有2種方式

## 過濾

Open-WebUI 的 `Filter` 類別就像是聊天過程中的**「訊息加工廠」**，主要負責三個動作：

1. **攔截輸入 (`inlet`)**：
在你的問題傳給 AI **之前**，先攔截下來修改。
*(範例：這支程式就是在這裡把你的「中文問題」偷換成「英文問題」才送出去。)*
2. **攔截輸出 (`outlet`)**：
在 AI 的回答傳回給你 **之後**，再攔截下來加工。
*(範例：可以在 AI 的回答後面自動加上「此回答僅供參考」的警語。)*
3. **開關控制 (`Valves`)**：
提供一個視覺化的開關，讓使用者可以在網頁上直接開啟或關閉這些功能。

**一句話總結：**
它讓你有權限在「你」與「AI」的對話中間**動手腳**，而不只是單純的傳話。
