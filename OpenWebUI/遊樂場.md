# 遊樂場

> **用來「直接測試模型能力與提示詞效果」的即時實驗區**

以下用**教學與實務角度**幫你清楚拆解它的用途。

---

## **一、Open WebUI「遊樂場」是什麼？**

**遊樂場（Playground）是一個讓你不寫程式、不建立工作流**，就能直接與模型互動的地方，用來：

- 測試模型是否「會不會答」
- 比較不同 Prompt 的差異
- 驗證 System Prompt 是否有效
- 驗證 RAG 是否真的有吃到文件
- 快速觀察模型幻覺（Hallucination）

👉 它是 **「實驗室」而不是「正式應用」**

---

## **二、遊樂場 vs 一般聊天室差在哪？**

| **功能面向**      | **一般聊天** | **遊樂場（Playground）** |
| ------------- | -------- | ------------------- |
| System Prompt | 固定或隱藏    | ✅ 可自行輸入             |
| 使用者 Prompt    | 單一輸入     | ✅ 可反覆測試             |
| 模型切換          | 有限       | ✅ 快速切換              |
| RAG 文件        | 有，但較隱性   | ✅ 明確指定              |
| 溫度 / Top-p    | 通常不可調    | ✅ 可調                |
| 教學與測試         | 不適合      | ⭐ 非常適合              |

---

## **三、遊樂場最常見的 6 個用途（非常適合教學）**

### **1️⃣ 測試**

### **System Prompt 是否有效**

例如你可以測試：

```other
你只能根據文件回答，若沒有資料請回答「文件中未提及」
```

👉 看模型會不會亂補

---

### **2️⃣ 測試**

### **RAG 是否真的有用到文件**

你可以問：

> 這張信用卡的年費是多少？

再問一題：

> 這張信用卡是否支援加密貨幣回饋？

👉 正確行為是第二題回答「文件中未提及」

---

### **3️⃣ 比較**

### **同一問題，不同 Prompt 的差異**

例如：

- Prompt A：

    >> 請說明信用卡年費

- Prompt B：

    >> 請條列方式說明年費與免年費條件

👉 觀察回答結構是否改善

---

### **4️⃣ 比較**

### **不同模型的理解能力**

在本地端（Ollama）環境很常見：

- llama3
- gemma
- qwen
- mistral

👉 同一題，看誰：

- 不亂編
- 能算對
- 能說「不知道」

---

### **5️⃣ 測試**

### **參數對幻覺的影響**

例如：

- Temperature = 0.1
- Temperature = 0.8

👉 高溫度時，模型是否更容易「自己發揮」

---

### **6️⃣ 教學生「Prompt Engineering」**

遊樂場非常適合當課堂示範：

- ❌ 問太籠統
- ✅ 問題加上條件、格式、限制

---

## **四、如果用在你目前的教學情境（很適合）**

以你目前在教的內容來看，遊樂場特別適合用來：

### **✅ RAG 教學流程**

1. 上傳信用卡英文說明文件
2. 設定 system prompt（禁止亂答）
3. 在遊樂場逐題測試問題
4. 觀察模型是否準確引用文件

👉 **學生可以「親眼看到 RAG 有沒有生效」**

---

### **✅ LLM 行為觀察課**

你可以讓學生實驗：

- 拿掉 system prompt → 開始亂答
- 加上限制 prompt → 回答變保守
- 沒文件 → 幻覺出現
