# AI Prompt 參考：純 JavaScript 專案

## 使用說明

當你手動建立好環境後，可以使用以下 prompt 讓 AI 幫你寫程式碼。

---

## Prompt 範例 1：基本結構

```
我已經建立了一個純 JavaScript 專案，結構如下：
- index.html（已設定 type="module"）
- main.js（主程式）
- utils.js（工具函數）

請幫我：
1. 在 utils.js 中建立一個計算函數（加、減、乘、除）
2. 在 main.js 中引入並使用這些函數
3. 在頁面上顯示計算結果
```

---

## Prompt 範例 2：DOM 操作

```
我有一個純 JavaScript 專案，使用 ES Module。

請幫我建立一個簡單的待辦事項功能：
1. 有一個輸入框和按鈕
2. 點擊按鈕後，將輸入的內容加入列表
3. 每個項目旁邊有刪除按鈕
4. 使用模組化方式組織程式碼（可以分成多個檔案）

環境資訊：
- 使用 ES Module（import/export）
- 不使用任何框架或工具
- 現代瀏覽器環境
```

---

## Prompt 範例 3：模組化組織

```
我有一個純 JavaScript 專案，想用模組化的方式組織程式碼。

請幫我建立：
1. data.js - 存放資料
2. utils.js - 工具函數
3. dom.js - DOM 操作相關函數
4. main.js - 主程式，整合以上模組

功能需求：建立一個簡單的計算機應用
- 可以輸入數字
- 可以選擇運算符（+、-、*、/）
- 顯示計算結果

環境限制：
- 純 JavaScript，無任何工具
- 使用 ES Module
- 現代瀏覽器
```

---

## Prompt 範例 4：錯誤處理

```
我有一個純 JavaScript 專案，使用 ES Module。

請幫我建立一個表單驗證功能：
1. 姓名（必填，至少 2 個字）
2. 電子郵件（必填，格式驗證）
3. 密碼（必填，至少 8 個字，包含大小寫和數字）
4. 提交按鈕

驗證規則：
- 即時驗證（輸入時顯示錯誤訊息）
- 提交前再次驗證
- 錯誤訊息顯示在對應欄位下方

請使用模組化方式，將驗證邏輯和 DOM 操作分開。
```

## 使用技巧

### 1. 明確說明環境

```
✅ 好的描述：
「我有一個純 JavaScript 專案，使用 ES Module，不使用任何框架或工具」

❌ 不好的描述：
「幫我寫一個網頁」
```

### 2. 說明檔案結構

```
✅ 好的描述：
「專案結構：
- index.html（已設定 type="module"）
- main.js（主程式）
- utils.js（工具函數）」

❌ 不好的描述：
「幫我寫程式碼」
```

### 3. 說明技術限制

```
✅ 好的描述：
「環境限制：
- 純 JavaScript，無任何工具
- 使用 ES Module
- 現代瀏覽器
- 不使用 npm 套件」

❌ 不好的描述：
「用最簡單的方式」
```

### 4. 說明需求細節

```
✅ 好的描述：
「功能需求：
1. 有一個輸入框和按鈕
2. 點擊按鈕後，將輸入的內容加入列表
3. 每個項目旁邊有刪除按鈕」

❌ 不好的描述：
「做一個待辦事項」
```

## 常見問題

### Q: AI 給的程式碼不符合我的環境怎麼辦？

**A**: 明確告訴 AI：
```
「這個專案不使用 npm，請使用純 JavaScript 和 ES Module，
不要使用 require 或任何需要打包工具的語法」
```

### Q: AI 給的程式碼太複雜怎麼辦？

**A**: 要求簡化：
```
「請簡化程式碼，只保留核心功能，不要使用複雜的設計模式」
```

### Q: AI 給的程式碼有錯誤怎麼辦？

**A**: 提供錯誤訊息：
```
「執行時出現錯誤：xxx
請修正這個問題」
```

## 進階 Prompt 範例

### 完整專案描述

```
我已經手動建立了一個純 JavaScript 專案，環境如下：

專案結構：
- index.html（已設定 type="module"）
- main.js（主程式）
- components/（元件資料夾）
- utils/（工具函數資料夾）

技術限制：
- 純 JavaScript，無任何工具
- 使用 ES Module（import/export）
- 現代瀏覽器環境
- 不使用 npm 套件
- 不使用框架

功能需求：
建立一個簡單的部落格文章列表：
1. 從 data.js 讀取文章資料
2. 顯示文章標題、作者、日期
3. 點擊文章可以展開顯示完整內容
4. 可以搜尋文章（標題或內容）

請幫我：
1. 設計檔案結構
2. 寫出所有必要的程式碼
3. 使用模組化方式組織
4. 加上適當的註解
```

---

## 提示

1. **先建立環境**：手動建立好檔案結構後，再請 AI 寫程式碼
2. **明確說明限制**：告訴 AI 你的環境限制（無工具、純 JavaScript）
3. **逐步建立**：不要一次要求太多功能，可以分步驟
4. **檢查結果**：AI 給的程式碼要檢查是否符合你的環境

---

**記住：AI 是工具，理解環境配置才是重點！**
