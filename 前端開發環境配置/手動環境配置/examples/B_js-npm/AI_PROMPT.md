# AI Prompt 參考：JavaScript + npm 專案

## 使用說明

這個範例是**故意失敗**的，目的是讓學生理解「瀏覽器無法直接使用 npm 套件」的問題。

但當你理解問題後，可以使用以下 prompt 讓 AI 幫你解決問題或建立替代方案。

---

## Prompt 範例 1：理解問題

```
我有一個 JavaScript + npm 專案，嘗試在瀏覽器中直接使用 lodash：

```javascript
import _ from 'lodash';
```

但瀏覽器報錯：
"Failed to resolve module specifier 'lodash'"

請幫我：
1. 解釋為什麼會出現這個錯誤
2. 說明瀏覽器為什麼無法直接使用 npm 套件
3. 提供解決方案（需要什麼工具）
```

---

## Prompt 範例 2：不使用 npm 套件的替代方案

```
我有一個純 JavaScript 專案（不使用任何打包工具），
原本想使用 lodash 的某些功能，但發現瀏覽器無法直接使用。

請幫我：
1. 不使用 lodash，用純 JavaScript 實作以下功能：
   - 陣列去重（uniq）
   - 陣列分組（groupBy）
   - 物件深拷貝（cloneDeep）
2. 使用 ES Module 組織程式碼
3. 確保可以在現代瀏覽器中執行

環境限制：
- 純 JavaScript
- 使用 ES Module
- 不使用任何 npm 套件
- 不使用打包工具
```

---

## Prompt 範例 3：理解工具的作用

```
我理解瀏覽器無法直接使用 npm 套件，需要打包工具。

請幫我：
1. 解釋打包工具（Bundler）做了什麼事
2. 說明為什麼需要打包工具
3. 列出常見的打包工具（Webpack、Vite、Rollup）
4. 比較它們的差異

我的專案情況：
- 想使用 npm 套件（例如 lodash）
- 使用 ES Module
- 需要在瀏覽器中執行
```

---

## Prompt 範例 4：手動下載套件（不推薦但可行）

```
我有一個純 JavaScript 專案，無法使用打包工具，
但想使用 lodash 的功能。

請幫我：
1. 說明如何手動下載 lodash 的 ES Module 版本
2. 如何手動引入到專案中
3. 這種方式的優缺點

環境限制：
- 不使用 npm
- 不使用打包工具
- 純 JavaScript + ES Module
```

---

## 教學重點

這個範例的目的是：

1. **體驗問題**：讓學生親身體驗「瀏覽器無法直接使用 npm 套件」
2. **理解原因**：理解為什麼會出現這個問題
3. **引出解決方案**：自然引出「需要打包工具」的概念

---

## 使用建議

### 階段 1：體驗問題

先不要用 AI，自己嘗試：
1. 執行 `npm install lodash`
2. 嘗試在瀏覽器中 `import _ from 'lodash'`
3. 觀察錯誤訊息
4. 思考為什麼會失敗

### 階段 2：理解問題

使用 AI 幫助理解：
```
「為什麼瀏覽器無法直接使用 npm 套件？
請詳細解釋原因和解決方案」
```

### 階段 3：尋找替代方案

如果暫時不想用打包工具，可以請 AI 提供替代方案：
```
「我不想使用打包工具，請用純 JavaScript 實作 xxx 功能」
```
