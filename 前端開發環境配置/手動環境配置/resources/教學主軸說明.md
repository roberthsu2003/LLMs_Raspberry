# 教學主軸說明

## 核心教學主軸

> **前端環境 = 語言 + 編譯 + 模組系統 + 開發工具 + 框架**

這是整份教學的核心觀念。只要學生能分清楚這五件事，後面不管是 Vite、Webpack、React、Vue，都不會怕。

---

## 五個層次詳解

### 1. 語言（Language）

**定義**：程式碼的語法本身

**範例**：
- JavaScript
- TypeScript
- JSX（雖然不是語言，但需要特別處理）

**特點**：
- 這是基礎層
- 決定你寫什麼語法
- 瀏覽器最終執行的是 JavaScript

**教學重點**：
- JavaScript 是基礎
- TypeScript 是 JavaScript 的超集
- JSX 需要轉換成 JavaScript

---

### 2. 編譯（Compilation）

**定義**：把一種語法轉換成另一種語法

**範例**：
- TypeScript → JavaScript（tsc）
- JSX → JavaScript（Babel、Vite）
- ES6+ → ES5（Babel）

**特點**：
- 轉換過程
- 開發時的工具
- 瀏覽器不認識原始語法

**教學重點**：
- TypeScript 永遠不會在瀏覽器執行
- 編譯是轉換過程
- 編譯後的程式碼才是瀏覽器執行的

---

### 3. 模組系統（Module System）

**定義**：如何組織和載入程式碼

**範例**：
- ES Module（`import/export`）
- CommonJS（`require/module.exports`）
- npm 套件管理

**特點**：
- 程式碼組織方式
- 如何載入其他檔案
- 如何管理依賴

**教學重點**：
- 現代瀏覽器支援 ES Module
- npm 是開發階段的工具
- 瀏覽器無法直接使用 npm 套件

---

### 4. 開發工具（Development Tools）

**定義**：幫助開發的工具

**範例**：
- Vite
- Webpack
- 開發伺服器
- 熱更新（HMR）

**特點**：
- 改善開發體驗
- 整合其他工具
- 開發時使用

**教學重點**：
- 工具是為了改善開發體驗
- 不是所有專案都需要
- 可以選擇不同的工具

---

### 5. 框架（Framework）

**定義**：解決特定問題的程式庫

**範例**：
- React（UI 問題）
- Vue（UI 問題）
- Angular（完整框架）

**特點**：
- 解決特定問題
- 提供開發模式
- 可選的

**教學重點**：
- 框架解決特定問題
- 不是所有專案都需要框架
- 可以選擇不同的框架

---

## 教學策略

### 從下往上學習

1. **先學語言層**（JavaScript）
2. **再學編譯層**（TypeScript）
3. **再學模組系統**（npm、ES Module）
4. **再學開發工具**（Vite）
5. **最後學框架**（React）

### 每層都要理解

- **這層解決什麼問題？**
- **沒有這層會怎樣？**
- **這層的輸入和輸出是什麼？**

### 建立對照基準

- **沒有工具時**：瀏覽器原生能力
- **有工具時**：工具提供的額外功能
- **對照差異**：理解工具的價值

---

## 常見問題

### Q: 一定要學全部五層嗎？

**A**: 不一定。根據專案需求選擇：
- 簡單專案：可能只需要語言層
- 中型專案：語言 + 模組系統 + 開發工具
- 大型專案：完整的五層

### Q: 可以跳過某些層嗎？

**A**: 可以，但建議至少理解：
- 語言層（必須）
- 模組系統（幾乎必須）
- 其他層根據需求

### Q: 這五層的順序可以改變嗎？

**A**: 學習順序可以調整，但建議按照「從下往上」的順序，因為上層依賴下層。

---

## 教學目標

完成教學後，學生應該能夠：

1. ✅ 清楚說出每一層的作用
2. ✅ 理解每一層解決的問題
3. ✅ 知道如何組合這些層
4. ✅ 能夠判斷何時需要哪一層
5. ✅ 能夠手動配置一個專案
