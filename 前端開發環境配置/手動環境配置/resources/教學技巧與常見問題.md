# 教學技巧與常見問題

## 教學技巧

### 每章的使用方式

你可以在每一章：

1. **開頭放 🎯 教學目標**
   - 告訴學生「這堂課要幹嘛」
   - 讓學生有明確的學習方向

2. **中間教內容**
   - 按照章節內容詳細講解
   - 實際操作示範

3. **結尾用 ⚠️ 學生常見迷思當 QA 討論**
   - 預先回答學生可能有的疑問
   - 讓學生覺得「老師早就知道我會卡在哪」

## 課堂關鍵提問

在教學過程中，可以問這些問題：

### 理解檢查

- 「現在瀏覽器實際跑的是哪個檔案？」
- 「這一層拿掉會發生什麼事？」
- 「誰在幫誰工作？」

### 思考引導

- 「為什麼需要這個工具？」
- 「沒有這個工具會怎樣？」
- 「這個工具解決了什麼問題？」

### 應用檢查

- 「你現在最困擾的是什麼？」
- 「這個工具如何解決你的問題？」
- 「什麼時候該用這個工具？」

## 常見學生提問（FAQ）

### 關於工具的必要性

#### Q1：一定要用這些工具嗎？

**A**：不一定。根據專案需求：
- 簡單專案：可能不需要
- 複雜專案：幾乎都需要
- 學習階段：建議先理解原理

#### Q2：不用工具就不能寫前端嗎？

**A**：可以，但會很痛苦：
- 無法使用 npm 套件
- 無法使用 TypeScript/JSX
- 開發體驗差

#### Q3：一開始就該用工具嗎？

**A**：學習原理時不該，用來做專案時該：
- 學習階段：理解原理
- 實作階段：使用工具

---

### 關於 TypeScript

#### Q4：一定要學 TypeScript 嗎？

**A**：不一定，但建議學：
- 現代前端幾乎都用 TypeScript
- 型別安全很重要
- 更好的開發體驗

#### Q5：TypeScript 很難嗎？

**A**：基本使用不難：
- 基本的型別註解很簡單
- 有型別推斷，很多時候不需要明確寫型別
- 進階功能可以慢慢學

#### Q6：TypeScript 會讓程式跑得比較快嗎？

**A**：不會。TypeScript 只是開發時的工具：
- 執行的是編譯後的 JavaScript
- 速度一樣
- 但開發時會更安全

---

### 關於 React

#### Q7：一定要用 React 嗎？

**A**：不一定。還有其他選擇：
- Vue
- Angular
- 甚至不用框架（純 JavaScript）

#### Q8：學會 React 就不用學 JavaScript 了嗎？

**A**：不行。React 建立在 JavaScript 之上：
- 需要 JavaScript 基礎
- React 只是工具，JavaScript 是基礎

#### Q9：React 就是在寫 HTML 嗎？

**A**：不是。React 使用 JSX：
- 看起來像 HTML，但不是
- JSX 需要轉換成 JavaScript
- 最終執行的是 JavaScript

---

### 關於 Vite

#### Q10：Vite 就是新的 React 嗎？

**A**：不是。Vite 是工具，React 是框架：
- Vite：解決開發流程問題
- React：解決 UI 問題
- 兩者不同層次

#### Q11：不用 Vite 就不能寫現代前端嗎？

**A**：可以。還有其他選擇：
- Webpack
- Rollup
- Parcel
- 甚至不用打包工具（簡單專案）

#### Q12：Vite 只是拿來跑起來比較快嗎？

**A**：不只是。Vite 提供完整的工具鏈：
- 開發伺服器
- 打包工具
- 熱更新
- 工具整合

---

### 關於學習順序

#### Q13：可以跳過某些章節嗎？

**A**：不建議。每章都建立在前一章的基礎上：
- 循序漸進很重要
- 跳過可能導致不理解

#### Q14：一定要按順序學嗎？

**A**：建議按順序，但可以：
- 有基礎的可以跳過簡單的章節
- 遇到問題時回頭看前面的章節

#### Q15：多久可以學完？

**A**：根據個人情況：
- 有基礎：1-2 週
- 初學者：1-2 個月
- 重點是理解，不是速度

## 教學節奏建議

### 整體節奏

1. **先不用工具，讓學生痛一次**
   - 體驗手動編譯
   - 體驗手動重新整理
   - 理解為什麼需要工具

2. **問學生：「你現在最困擾的是什麼？」**
   - 收集痛點
   - 了解需求

3. **再引入工具**
   - 展示工具如何解決問題
   - 對照差異

4. **回頭對照**
   - 讓學生看到具體的差異
   - 理解工具的價值

### 每章節奏

1. **說明教學目標**（5 分鐘）
   - 告訴學生這章要學什麼

2. **講解核心觀念**（20-30 分鐘）
   - 詳細解釋概念
   - 實際操作示範

3. **學生實作**（20-30 分鐘）
   - 讓學生跟著做
   - 即時解答問題

4. **討論常見迷思**（10 分鐘）
   - 回答學生疑問
   - 加深理解

## 除錯技巧

### 常見錯誤處理

#### 錯誤 1：模組找不到

**症狀**：`Cannot find module 'xxx'`

**解決**：
1. 檢查 `package.json` 是否有安裝
2. 執行 `npm install`
3. 檢查路徑是否正確

#### 錯誤 2：型別錯誤

**症狀**：TypeScript 編譯錯誤

**解決**：
1. 檢查型別是否正確
2. 查看錯誤訊息
3. 使用型別斷言（必要時）

#### 錯誤 3：瀏覽器報錯

**症狀**：瀏覽器 Console 有錯誤

**解決**：
1. 查看錯誤訊息
2. 檢查檔案路徑
3. 檢查語法是否正確

## 評估方式

### 理解檢查

- 能否說出每一層的作用？
- 能否解釋為什麼需要這個工具？
- 能否判斷何時該用這個工具？

### 實作檢查

- 能否手動配置一個專案？
- 能否解決常見問題？
- 能否選擇適合的工具？

### 應用檢查

- 能否建立一個實際專案？
- 能否除錯和解決問題？
- 能否選擇適合的工具？

## 總結

教學的重點不是「教會學生用工具」，而是：

1. ✅ **理解原理**：知道為什麼需要工具
2. ✅ **選擇工具**：能夠選擇適合的工具
3. ✅ **解決問題**：遇到問題時知道如何除錯
4. ✅ **持續學習**：能夠自主學習新工具

**記住：工具是手段，理解原理才是目的！**
