## 範例 A：純 JavaScript（無任何工具）

### 技術組合
- 純 JavaScript
- ES Module
- 無任何工具

### 教學重點
- 瀏覽器原生能力
- ES Module 的使用
- 沒有任何工具時的限制

### 檔案結構
```
A_js-basic/
├── index.html
├── main.js
└── msg.js
```

### 如何執行
1. 直接用瀏覽器開啟 `index.html`
2. 或使用本地伺服器：`python3 -m http.server 8000`

### 學習目標
- 理解瀏覽器原生支援 ES Module
- 知道沒有任何工具時的限制
- 建立基準線

---

## 範例 B：JavaScript + npm（不打包）

### 技術組合
- JavaScript
- npm
- 不打包（會失敗）

### 教學重點
- npm 套件管理
- 瀏覽器無法直接使用 npm 套件
- 自然引出「需要 Bundler」的概念

### 檔案結構
```
B_js-npm/
├── index.html
├── main.js
├── package.json
└── node_modules/
```

### 如何執行
1. `npm install`
2. 嘗試在瀏覽器中開啟（會失敗）
3. 觀察錯誤訊息

### 學習目標
- 理解 npm 的角色
- 知道瀏覽器無法直接使用 npm 套件
- 理解為什麼需要打包工具

**注意**：這個範例「故意失敗」，目的是讓學生理解問題。

## 範例 C：TypeScript 基礎（無框架、無 Vite）

### 技術組合
- TypeScript
- tsc（TypeScript Compiler）
- 無框架、無 Vite

### 教學重點
- TypeScript 編譯流程
- tsc 的作用
- TypeScript → JavaScript 的轉換

### 檔案結構
```
C_ts-basic/
├── src/
│   └── main.ts
├── dist/
│   └── main.js
├── tsconfig.json
└── package.json
```

### 如何執行
1. `npm install`
2. `npm run build`（編譯 TypeScript）
3. 查看 `dist/main.js`（編譯後的 JavaScript）
4. 在瀏覽器中開啟 HTML 檔案

### 學習目標
- 理解 TypeScript 編譯流程
- 知道 TypeScript 永遠不會在瀏覽器執行
- 理解 tsc 的作用

## 範例 F：React + TypeScript + Vite（完整現代環境）

### 技術組合
- React
- TypeScript
- Vite
- 完整的工具鏈

### 教學重點
- 整合所有概念
- 每一層技術的作用
- 現代前端專案的標準組成

```
F_react-ts-vite/
├── index.html
├── src/
│   ├── App.tsx
│   ├── main.tsx
│   └── components/

├── vite.config.ts
├── tsconfig.json
└── package.json
```

### 如何執行
1. `npm install`
2. `npm run dev`（開發）
3. `npm run build`（建置）

### 學習目標
- 理解完整環境的組成
- 知道每一層的作用
- 能夠手動配置完整專案

## 學習路徑建議

### 循序漸進

1. **從 A 開始**：理解瀏覽器原生能力
2. **到 B**：體驗 npm 套件的矛盾
3. **到 C**：理解 TypeScript 編譯
4. **到 D**：體驗 Vite 的便利
5. **到 E**：理解 React 需要工具
6. **到 F**：整合所有概念

### 對照學習

- 每完成一個範例，對照前一個範例
- 思考「增加了什麼？」「解決了什麼問題？」
- 理解工具的價值

## 使用建議

### 給學生

1. **按順序執行**：不要跳過任何範例
2. **實際動手**：不要只看，要實際執行
3. **觀察差異**：對照前後範例的差異
4. **思考問題**：每完成一個範例，思考「為什麼需要這個？」
5. **使用 AI 協助**：手動建立環境後，參考 `AI_PROMPT.md` 使用 AI 寫程式碼

### 給老師

1. **示範執行**：每個範例都要示範執行
2. **引導思考**：讓學生思考「增加了什麼？」
3. **對照差異**：對照前後範例的差異
4. **解答疑問**：及時解答學生的疑問

## 常見問題

### Q1：範例 B 為什麼會失敗？

**A**：這是故意的。目的是讓學生理解「瀏覽器無法直接使用 npm 套件」的問題。

### Q2：可以跳過某些範例嗎？

**A**：不建議。每個範例都建立在前一個的基礎上，跳過可能導致不理解。

### Q3：範例可以修改嗎？

**A**：可以！鼓勵學生修改和實驗，這樣更能理解。

## 延伸練習

完成所有範例後，可以嘗試：

1. **修改範例**：嘗試修改程式碼，看看會發生什麼
2. **組合範例**：嘗試組合不同範例的功能
3. **建立專案**：用學到的知識建立一個實際專案
4. **使用 AI**：參考 `AI_PROMPT.md`，使用 AI 協助建立更複雜的功能

## 🤖 關於 AI Prompt

每個範例專案都包含 `AI_PROMPT.md` 檔案，提供：

- **Prompt 範例**：如何向 AI 描述你的環境和需求
- **使用技巧**：如何確保 AI 給的程式碼符合你的環境
- **常見問題**：遇到問題時如何修正
- **進階範例**：完整的專案描述範例

**重要**：
- 本教學的重點是「手動建立環境」
- 程式碼部分可以使用 AI 協助
- 但要理解 AI 給的程式碼，不要只是複製
